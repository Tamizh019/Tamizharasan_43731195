Model Layer (Entities)

User Entity

Field Type Description
id Long Primary key (auto-generated)
username String Unique username
email String Unique email
password String BCrypt-hashed password
role Role Enum: ADMIN MODERATOR USER
isOnline boolean Real-time online status
isBanned Boolean Admin ban flag
avatarStyle String Avatar display style ("initials", "uploaded")
lastSeen LocalDateTime Last activity timestamp
bannedAt LocalDateTime When user was banned
bannedBy String Admin who banned the user
createdAt LocalDateTime Account creation timestamp

Message Entity

Field Type Description
id Long Primary key
type MessageType CHAT JOIN LEAVE
content String Message text
sender String Username of sender
timestamp LocalDateTime Auto-set on persist

SharedFile Entity

Stores uploaded files as BLOBs directly in the database.

Field Type Description
id Long Primary key
originalFilename String User-uploaded name
contentType String MIME type
fileSize Long Size in bytes
fileData byte[] Actual file binary (BLOB)
uploadedBy String Username

Service Layer (Business Logic)

AuthService

login(LoginRequest)

Authenticates user via AuthenticationManager, returns JWT token


register(RegisterRequest)

Validates Supabase token, creates user with hashed password


validateSupabaseToken()

Calls Supabase API to verify email verification token

GeminiService

generateWithTools()

Sends prompt to Gemini API with function declarations


buildToolDeclarations()

Defines available functions: get_users, get_files, get_messages


parseResponse()

Extracts text or function call from Gemini JSON response

KnowledgeService

Provides PDF RAG (Retrieval-Augmented Generation) by reading PDF files from the knowledge/ folder.


Enables the AI assistant to access and reference documentation stored in PDF format.

CustomUserDetailsService

loadUserByUsername()

Implements Spring Security's UserDetailsService interface


Loads user from database with roles for authentication

Controller Layer (REST APIs)

AuthController /api/auth

Endpoint Method Description
/login POST Authenticate user, return JWT
/register POST Create new user account

ChatController /api/chat

Endpoint Method Description
/history GET Fetch all messages (ordered by timestamp)
/online GET Get list of currently online usernames
/{id} DELETE Delete a message (role-based permissions)

WebSocket Mappings

  • /app/chat.sendMessage → Broadcast to /topic/public
  • /app/chat.addUser → User join handler

AiController /api/ai

Endpoint Method Description
/test GET Test Gemini connection
/chat POST Main chat endpoint with Function Calling (MCP)

AdminController /api/admin

Endpoint Method Description
/stats GET Dashboard stats (users, messages, files)
/stats/activity GET Message activity for last 7 days (chart data)
/users GET List all users with details
/users/{id}/ban POST Ban a user
/users/{id}/promote POST Promote to Moderator
/users/{id}/unban POST Unban a user
/users/{id}/demote POST Demote Moderator to User
/users/{id}/messages GET View user's message history (Spy Mode)

FileController /api/files

Endpoint Method Description
/upload POST Upload a new file (max 50MB)
/ GET List all shared files (metadata only)
/{id} GET Download a file by ID
/{id}/info GET Get file metadata
/{id} DELETE Delete a file (owner only)

UserController /api/users

Endpoint Method Description
/ GET Get all users (public list)
/{username}/avatar GET Get user's avatar image
/avatar POST Upload avatar image
/avatar DELETE Delete own avatar
/{username}/stats GET Get user statistics (message count, join date)
/profile GET Get current user profile
/profile PUT Update profile (username, avatar style)

WebSocket & Real-Time Communication

Configuration

// STOMP endpoint (with SockJS fallback) registry.addEndpoint("/ws").setAllowedOriginPatterns("*").withSockJS(); // Message broker registry.setApplicationDestinationPrefixes("/app"); registry.enableSimpleBroker("/topic");

Message Flow

Client

Sends message

→

Server

Processes & validates

→

Broker

Broadcasts to subscribers

→

All Clients

Receive update

WebSocket Events

CONNECT
Client establishes WebSocket connection to /ws
SUBSCRIBE
Client subscribes to /topic/public for broadcasts
SEND
Client sends message to /app/chat.sendMessage
DISCONNECT
Server updates online status and notifies other users

AI Integration (Sparky)

Sparky uses Google Gemini Pro with Function Calling to interact with application data, providing intelligent responses based on real-time system information.

Available Functions (MCP Tools)

Function Parameters Description
get_users filter (optional: "online", "offline", "all") Returns user list with status
get_files limit (optional) Returns recent shared files
get_messages limit, sender (optional) Returns chat history
